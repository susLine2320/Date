# 日付表示・空調制御プラグイン
このプラグインは日付情報の設定および空調のシミュレートをするプラグイン。

## 1. 概要
- 現実の時間・季節に連動した「日付表示機能」
- 特定の年・月・曜日に固定する機能
- 上記の月・シナリオ内時間に応じた「外気温設定機能」
- 外気温に応じフィッティング制御を行う「空調機能」

を備えています。

#### 607番地上子補足
- 10以下（曜日のみ）設定…年は現在年が設定される。
- 607番地上子を置くと指定の年・月・曜日で現在日付から近い日が自動で設定される。
- 607番地上子を置かない・あるいは任意番号を0とすると現在の日付が設定される。
- 日付変更は未検証であることに留意すること。

### signalインデックス
なし

### SWインデックス
以下の設定ファイルで変更できます。デフォルトは`+`キー、`*`キー（いずれもテンキーではない）です。

設定したキー（Next/Prev）により、運転台のACスイッチを操作します。
* **右回し (Next)**: 切 → 入 → 自動 （自動で停止）
* **左回し (Prev)**: 自動 → 入 → 切 （切で停止）

### 設定ファイル
`Ats.dll` と同階層に配置してください。キーボードの操作割り当てを変更できます。

```ini
[Control]
; 操作キーの仮想キーコード (10進数)
; 186 = '+'キー / 187 = '*'キー
AC_Key_Next = 186
AC_Key_Prev = 187
```

## 3.補足事項
  * **スイッチの挙動**: 右回し（Next）は「切→入→自動」で止まり、左回し（Prev）は「自動→入→切」で止まります。ループはしません。
  * **温度変化の特性**: 地上子(`#608`)で指定した温度には、毎秒約0.1～0.3度のペースで物理的に計算されたラグを伴って変化します。
  * **暖房時の静音化**: 実車の仕様に基づき、**暖房モード（Mode 3）** 時は空調稼働音（Sound`#226`）が自動的に停止します。
  * **リセットの活用**: トンネル出口などに `Beacon.Put(608, 0, 0);` を置くことで、季節に合わせた外気温へ自然に復帰させることができます。

## 4.技術仕様
本プラグインにおける温度制御ロジックの技術的詳説を行う。本システムは「動的フィッティング」を基幹としている。

### 1. 外気温フィッティングの数理モデル

外気温の決定は、**「時系列ベース気温」**、**「地上子補正（オフセット）」**、**「動的ゆらぎ」** の3層構造で構成される。

#### A. ベース気温の算出
月ごとの平均気温 $T_{month}$ と、時刻 $h$ による日変化 $T_{time}$ を合成する。日変化は14時をピークとする正弦波（サインカーブ）で近似される。

$$T_{base} = T_{month} + T_{offset\_day} + 5.0 \cdot \sin\left(\frac{(h - 8.0) \cdot \pi}{12}\right)$$

ここで $T_{offset\_day}$ は、初期化時に決定される「その日の運勢（日次誤差）」であり、同じ月・時刻でもプレイごとに異なる環境を提供する。

#### B. オフセットの線形追従
地上子（`#608`）によって指定された目標外気温 $T_{target}$ に対し、現在の補正値 $T_{off\_now}$ を一定の速度で変化させる。

$$\Delta T_{target\_off} = T_{target} - T_{base}$$

現在のオフセット $T_{off\_now}$ は、経過時間 $\Delta t$（秒）に対し、以下の微分方程式に従って更新される。

$$\frac{dT_{off}}{dt} = \text{sgn}(\Delta T_{target\_off} - T_{off\_now}) \cdot 0.1$$

これにより、1秒間に $0.1^\circ\text{C}$ という一定の速度（スルーレート）で目標値へ収束する。これは、急激な外気温変化によるデジタル計のチャタリングを抑制し、環境の変化に「物理的な移動時間」の概念を持たせるためである。

### 2. センサーゆらぎ（Wobble）のシミュレーション

定常状態においてもデジタル計の末尾が微動する挙動は、ランダムウォーク（べき乗則ノイズに近い挙動）によって再現される。

$$W_{next} = \text{clamp}(W_{now} + \epsilon, -1.0, 1.0)$$
$$\epsilon \sim \text{Uniform}(-0.005, 0.005)$$

このゆらぎ $W$ を最終的な計算値に加算することで、回路のノイズやセンサーの分解能による微細な変動を表現している。

### 3. 車内温度の熱交換モデル

車内温度 $T_{room}$ は、外気温 $T_{outer}$ との熱交換、車内熱源（乗客・機器）、および空調能力の三要素によって決定される。

$$\frac{dT_{room}}{dt} = \alpha(T_{outer} - T_{room}) + \beta + \gamma(mode)$$

各係数の定義は以下の通りである。

* **自然熱交換係数 ($\alpha = 0.01$):** 車体の断熱性能に依存する。外気温との差に比例して温度が変化する。
* **内部熱源係数 ($\beta = 0.005$):** 乗客の体温や電気機器の発熱による定常的な温度上昇。
* **空調能力係数 ($\gamma$):** 冷房時は $-0.06$、暖房時は $+0.06$。送風・切時は $0$ となる。

このモデルにより、真夏の炎天下で冷房を切ると、車内温度が外気温を上回って上昇し続ける（ビニールハウス現象）といった非線形な挙動が自然に発生する。

---

### 4. 空調制御のヒステリシス（履歴特性）

「自動（AUTO）」モードにおける空調の頻繁なON/OFF（ハンチング現象）を防止するため、動作切替に境界幅（ヒステリシス）を持たせている。

* **冷房起動:** $T_{room} > 26.5^\circ\text{C}$
* **冷房停止:** $T_{room} < 25.0^\circ\text{C}$
* **暖房起動:** $T_{room} < 17.0^\circ\text{C}$
* **暖房停止:** $T_{room} > 19.0^\circ\text{C}$

この $1.5 \sim 2.0^\circ\text{C}$ の不感帯（デッドバンド）により、実車のオートエアコンと同様の、安定した稼働サイクルを実現している。
